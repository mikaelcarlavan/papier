Tables

The Table element draws a grid of rows and columns, wrapping cell text automatically and sizing rows to fit. TableCell overrides the table's defaults for any individual cell — its font, colours, alignment, padding, borders, and how many rows or columns it spans.

A first table

A table is created at a position, configured with chained setters, then filled row by row and added to a page.

use Papier\Elements\{Color, Table};

$regular = $doc->addFont('Helvetica');
$bold    = $doc->addFont('Helvetica-Bold');

$table = Table::create(36, 755)
    ->setColumnWidths(160, 70, 70, 70, 70)
    ->setFont($regular, 10, 'Helvetica')
    ->setHeaderFont($bold, 10, 'Helvetica-Bold')
    ->setHeaderRows(1)
    ->setHeaderBg(Color::rgb(0.12, 0.22, 0.42))
    ->setHeaderTextColor(Color::white())
    ->setCellPadding(5);

$table->addRow(['Category', 'Q1', 'Q2', 'Q3', 'Q4']);
$table->addRow(['Software Licenses', '1 240', '1 380', '1 510', '1 870']);
$table->addRow(['Hardware', '340', '290', '410', '530']);

$page->add($table);

The coordinates passed to Table::create() are the top-left corner. Rows then grow downward, toward decreasing y — the opposite direction from most elements, and deliberately so, because a table is read from the top and you almost always know where its top should sit.

Rows and cells

addRow() takes an array whose entries may be plain strings or TableCell objects, mixed freely. Strings inherit every table default; cells override selectively.

use Papier\Elements\TableCell;

$table->addRow(['Alice', 'Engineer', 'alice@example.com']);

$table->addRow([
    TableCell::make('Total')->align('left')->font($bold, 10, 'Helvetica-Bold'),
    TableCell::make('2 890')->font($bold, 10, 'Helvetica-Bold'),
    'unstyled',
]);

Building rows in a loop from an array of data is the normal case, and mixing forms lets you decorate only the cells that need it:

foreach ($rows as [$name, $uptime, $state]) {
    $table->addRow([
        $name,
        TableCell::make($uptime)->align('right'),
        TableCell::make($state)->align('center')->bg(Color::rgb(0.85, 0.95, 0.82)),
    ]);
}

Column widths

There are two ways to size columns, and the explicit one takes priority.

$table->setColumnWidths(160, 70, 70, 70, 70);   // five columns, in points
$table->setWidth(451);                          // total width, split equally

setColumnWidths() is variadic, and the number of arguments defines the column count. setWidth() is only consulted when no explicit widths were given; it distributes the total evenly. Give a text-heavy first column significantly more room than the numeric columns beside it — that is what makes the difference between a table that wraps sensibly and one that produces three-line cells everywhere.

Text style and wrapping

Cell text wraps automatically at word boundaries within the column width, and the row grows taller to accommodate however many lines result.

$table->setFont($regular, 10, 'Helvetica')
      ->setTextColor(Color::gray(0.15))
      ->setTextAlign('right')
      ->setVerticalAlign('top')
      ->setLineHeight(1.3);

The third argument to setFont() is the base font name used for character-width metrics. Supplying it — 'Helvetica', 'Times-Roman' — gives accurate AFM measurement and therefore correct line breaks and row heights. Leave it out with a standard font and the layout falls back to an estimate, which shows up as text that overflows its column or wraps a word too early.

Always pass the base font name when using one of the 14 standard fonts. Embedded TrueType fonts carry their own metrics and do not need the hint.

Horizontal alignment accepts 'left', 'center' and 'right'. Right-aligning numeric columns while leaving the label column left-aligned is worth the extra call on any table containing figures: it lines the digits up so magnitudes are comparable at a glance.

Row heights

$table->setRowHeight(24);      // fixed height for every row
$table->setMinRowHeight(32);   // floor; rows still grow for long content

By default rows size themselves to their content plus padding. setRowHeight() forces a fixed height, which gives a rigid grid but clips anything that does not fit. setMinRowHeight() is usually the better tool: it guarantees a comfortable minimum — useful for schedule grids and forms — while still letting a long cell expand.

Padding

Padding is the gap between the cell border and its text. Three setters cover increasing levels of control.

$table->setCellPadding(5);                       // all four sides
$table->setCellPaddingXY(8, 5);                  // horizontal, vertical
$table->setCellPaddingFull(5, 8, 5, 8);          // top, right, bottom, left

The default is 4 pt vertically and 6 pt horizontally. More horizontal than vertical padding is the conventional choice — it keeps columns visually separated without making rows unnecessarily tall.

Header rows

Declaring header rows tells the table that the first N rows are headings, so they can take their own font, background and text colour.

$table->setHeaderRows(1)
      ->setHeaderFont($bold, 10, 'Helvetica-Bold')
      ->setHeaderBg(Color::rgb(0.12, 0.22, 0.42))
      ->setHeaderTextColor(Color::white());

$table->addRow(['Category', 'Q1', 'Q2', 'Q3', 'Q4']);   // styled as a header

The header rows are simply the first rows you add — there is no separate method for them. Set setHeaderRows(2) when a table needs a two-level heading.

Footer rows

Footer rows work symmetrically at the bottom of the table, and are the right way to style a totals row.

$table->setFooterRows(1)
      ->setFooterFont($bold, 10, 'Helvetica-Bold')
      ->setFooterBg(Color::rgb(0.20, 0.42, 0.25))
      ->setFooterTextColor(Color::white());

// … body rows …

$table->addRow(['Total', '1 070 000', '853 000', '217 000']);   // the footer

The last N rows added are treated as the footer, so the call order stays natural: configure, add the body, add the totals last.

Row backgrounds

$table->setRowBg(Color::white())
      ->setAltRowBg(Color::rgb(0.93, 0.95, 0.99));

setAltRowBg() turns on zebra striping: alternating body rows take the alternate colour while the rest use the base row colour. On tables wider than a few columns this measurably helps the eye track across a row, and it is the cheapest readability win available. Passing null to setRowBg() leaves rows transparent so the page shows through.

Borders

The outer border of the table and the inner grid lines are configured independently, which is what makes minimal and heavily ruled styles both easy.

$table->setBorder(Color::rgb(0.3, 0.35, 0.45), 0.5)       // outer frame
      ->setInnerBorder(Color::rgb(0.7, 0.75, 0.85), 0.3); // grid lines

$table->noBorder();        // remove the outer frame
$table->noInnerBorder();   // remove the grid lines

Calling both noBorder() and noInnerBorder() produces a borderless list — the modern look, where alternating backgrounds and generous padding carry the structure instead of rules. Where you do want rules, making the inner lines lighter and thinner than the outer frame, as above, keeps the grid from competing with the data.

Per-cell and per-side borders

Individual cells can override the border colour and width, or suppress specific sides.

TableCell::make('Total')->border(Color::black(), 1.0);

TableCell::make('Total')->borderSides(null, false, null, false);   // no left/right

TableCell::make('Note')->noBorder();

borderSides() takes four nullable booleans in the order top, right, bottom, left. null means "inherit the table's setting", true forces the side on, and false suppresses it. Dropping the vertical rules across a totals row, as in the first pattern above, visually fuses the cells into a single summary bar without needing a colspan.

Spanning cells

Colspan

$table->addRow([
    TableCell::make('Engineering')
        ->colspan(5)
        ->align('center')
        ->bg(Color::rgb(0.78, 0.9, 0.78))
        ->font($bold, 10, 'Helvetica-Bold'),
]);

A cell spanning every column makes an in-table section divider, which is a cleaner way to group a long list than splitting it into several tables. The row array only needs the cells that actually exist — the spanned columns are not listed.

Rowspan

$table->addRow([
    TableCell::make('09:00')->align('center'),
    TableCell::make('Team Stand-up')->align('center')->valign('middle'),
    TableCell::make("Lab / Experiments\n(all morning)")
        ->rowspan(3)->align('center')->valign('middle'),
]);

$table->addRow([
    TableCell::make('09:30')->align('center'),
    TableCell::make('Sprint Planning')->align('center'),
    // column 2 is occupied by the rowspan above — omit it
]);

Rows following a rowspan must omit the cells covered by the span. Including them shifts every subsequent cell in that row one column to the right.

Explicit newlines in cell text are honoured alongside automatic wrapping, so "Lab / Experiments\n(all morning)" breaks where you asked.

Vertical alignment

$table->setVerticalAlign('top');                      // table-wide default

TableCell::make('Centred')->valign('middle');
TableCell::make('Bottom')->valign('bottom');

Accepted values are 'top', 'middle' and 'bottom'. Vertical alignment only becomes visible when a row is taller than its content — because a neighbouring cell wrapped to several lines, because of a rowspan, or because a minimum row height was set. Centring merged rowspan cells is the usual reason to reach for it.

Per-cell colours and fonts

Any cell can override the table's background, text colour and font. Encoding status through colour is the classic use:

$statusColor = match ($state) {
    'OK'       => Color::rgb(0.1, 0.55, 0.1),
    'DEGRADED' => Color::rgb(0.7, 0.5, 0.0),
    default    => Color::rgb(0.75, 0.1, 0.1),
};

$table->addRow([
    $service,
    TableCell::make($uptime)->align('right'),
    TableCell::make($state)
        ->align('center')
        ->bg(Color::rgb(0.98, 0.84, 0.84))
        ->color($statusColor)
        ->font($bold, 9, 'Helvetica-Bold'),
]);

See Text & fonts for the Color constructors.

Opacity

$ghost = Table::create(36, 752)
    ->setColumnWidths(160, 80, 80, 80)
    ->setFont($regular, 10, 'Helvetica')
    ->opacity(0.25);

opacity() applies to the whole table — borders, backgrounds and text alike. A faded table behind the real content makes a convincing "DRAFT" watermark that still reads as tabular data.

Worked example

examples/14_table.php builds a six-page document covering everything above: a striped sales table, colspan section dividers in an employee directory, a borderless feature list with wrapped multi-line cells, two side-by-side dashboards with per-cell status colours, a schedule grid using rowspan and all three vertical alignments, and a budget table with a styled footer row, a 25 % opacity watermark table, and suppressed per-cell border sides.

examples/16_table_utf8_alignment.php covers accented and multi-byte text in cells, where correct width measurement matters most.
