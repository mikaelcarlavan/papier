Annotations

Annotations are objects that live alongside a page's content rather than inside it — sticky notes, hyperlinks, highlights, review markup. Because they sit in a separate layer, a viewer can show, hide, edit, or export them without touching the page description underneath.

The annotation model

Every annotation is a dictionary attached to a page's /Annots array. It has a subtype that determines its behaviour, a rectangle that positions it, and a set of optional properties shared by all subtypes.

In Papier each subtype is a class extending the abstract Papier\Annotation\Annotation. The constructor always takes the bounding rectangle, and you attach the finished object to a page:

use Papier\Annotation\TextAnnotation;
use Papier\Elements\Color;

$note = new TextAnnotation(500, 745, 520, 765);
$note->setContents('Please review this section.')
     ->setIcon('Note')
     ->setColor(Color::rgb(1.0, 0.9, 0.0));

$page->addAnnotation($note);

The four constructor arguments are $x1, $y1, $x2, $y2 — the lower-left and upper-right corners in points, measured from the bottom-left of the page. This becomes the /Rect entry.

The rectangle is fixed at construction time. setRect() exists on the base class but is protected, so if you need different geometry, construct a new annotation rather than trying to move an existing one.

The subtypes

Fourteen subtypes cover the annotations you will normally author. They fall into four families: notes and links, text markup, geometric markup, and stamps.

LinkAnnotation/LinkA clickable region that jumps to a destination or opens a URI.
FreeTextAnnotation/FreeTextText drawn directly on the page rather than hidden in a pop-up.
HighlightAnnotation/HighlightMarks a text run with a translucent colour wash.
UnderlineAnnotation/UnderlineDraws a line beneath a text run.
StrikeOutAnnotation/StrikeOutDraws a line through a text run to mark it for deletion.
SquigglyAnnotation/SquigglyA wavy underline, conventionally flagging a spelling or usage problem.
LineAnnotation/LineA straight line with optional arrowheads at either end.

The src/Annotation/ directory also contains subtypes serving other features rather than page markup: PopupAnnotation (the pop-up window belonging to a markup annotation), WidgetAnnotation (form fields — see Forms), ScreenAnnotation, SoundAnnotation and MovieAnnotation (see Layers & multimedia), plus FileAttachmentAnnotation, CaretAnnotation, PolyLineAnnotation, WatermarkAnnotation, PrinterMarkAnnotation, TrapNetAnnotation and ThreeDAnnotation.

Common properties

Everything on the base class works on every subtype.

Contents and colour

setContents() supplies the annotation's text. For a sticky note this is the comment body; for markup annotations it is the reviewer's remark shown in the pop-up. It is also what assistive technology reads aloud, so fill it in even for purely visual annotations.

$annot->setContents('Figure 3 needs a caption.');

setColor() sets /C, the annotation's characteristic colour — the note icon's fill, the highlight wash, the line stroke. It takes a Color in any colour space:

use Papier\Elements\Color;

$annot->setColor(Color::rgb(1.0, 1.0, 0.0));
$annot->setColor(Color::hex('#ff0000'));
$annot->setColor(Color::cmyk(0, 0.5, 1.0, 0));
$annot->setColor(Color::gray(0.5));

$annot->clearColor();   // remove /C entirely — viewer default applies

Borders

setBorderStyle() writes the modern /BS dictionary: a width in points, a style character, and an optional dash pattern.

$annot->setBorderStyle(1.5);                    // 1.5 pt solid
$annot->setBorderStyle(1.0, 'D', [3, 2]);      // dashed: 3 on, 2 off
$annot->setBorderStyle(0);                     // no visible border

The style characters are S (solid, the default), D (dashed), B (bevelled), I (inset) and U (underline only). Setting the width to 0 is the usual way to suppress the blue rectangle viewers draw around links.

setBorder() writes the older /Border array with horizontal and vertical corner radii, for viewers that predate /BS.

Flags

setFlags() sets the /F bit field, which controls how a viewer treats the annotation. There are no named constants — pass the bits directly:

BitValueNameMeaning
10x001InvisibleHide if the subtype is unknown to the viewer
20x002HiddenNever display or print
30x004PrintInclude in printed output
40x008NoZoomDo not scale with the page
50x010NoRotateDo not rotate with the page
60x020NoViewPrint but do not display on screen
70x040ReadOnlyDisplay but forbid interaction
80x080LockedForbid deleting or changing properties
90x100ToggleNoViewInvert visibility on selection
100x200LockedContentsForbid changing the contents
 
$annot->setFlags(4);          // Print
$annot->setFlags(4 | 0x40);   // Print and ReadOnly

Annotations without the Print flag are invisible in printed output and in many rasterised previews. If an annotation must survive printing, set bit 3.

Other common setters

setName() writes /NM, a unique identifier for the annotation within its page — useful when a HideAction needs to address it. setModDate() records the modification time. setAction() attaches an action triggered on activation (see Bookmarks & navigation). setAppearance() supplies explicit normal, rollover and down appearance streams when you want full control over rendering.

Notes and links

A TextAnnotation is the classic sticky note. The rectangle sizes the clickable icon, not the pop-up.

use Papier\Annotation\TextAnnotation;

$note = new TextAnnotation(500, 745, 520, 765);
$note->setContents('This is a sticky-note annotation.')
     ->setIcon('Note')       // Comment, Key, Note, Help, NewParagraph, Paragraph, Insert
     ->setOpen(false)        // pop-up starts closed
     ->setColor(Color::rgb(1.0, 0.9, 0.0))
     ->setFlags(4);

setState() and setStateModel() record review status — for example the model Review with the state Accepted.

A LinkAnnotation makes a rectangle clickable. The simplest form opens a web address:

use Papier\Annotation\LinkAnnotation;

$link = new LinkAnnotation(72, 680, 272, 700);
$link->setURI('https://example.com')
     ->setHighlightMode('P')   // N=none, I=invert, O=outline, P=push
     ->setBorderStyle(0);      // suppress the default rectangle

A link has no appearance of its own — it is an invisible hotspot. Draw whatever the reader should see with ordinary page elements underneath it:

use Papier\Elements\{Color, Rectangle, Text};

$page->add(
    Rectangle::create(72, 680, 200, 20)->fill(Color::rgb(0.2, 0.4, 0.8)),
    Text::write('Click to visit example.com')->at(80, 686)->font($font, 11)->color(Color::white()),
);
$page->addAnnotation($link);

For links that jump within the document, use setDestination() with a destination array — see Bookmarks & navigation.

FreeTextAnnotation renders its contents on the page itself. Unlike other annotations it needs a default appearance string naming a font resource:

use Papier\Annotation\FreeTextAnnotation;

$free = new FreeTextAnnotation(72, 640, 300, 670);
$free->setContents('Text rendered directly on the page.')
     ->setDefaultAppearance('F1', 10)      // font RESOURCE NAME, not a font object
     ->setJustification(0)                 // 0 left, 1 centre, 2 right
     ->setColor(Color::rgb(0.9, 0.95, 1.0));

setCallout() adds a leader line from the text box to the thing it comments on, and setIntent() distinguishes a plain free text box from a callout or typewriter annotation.

Text markup

The four markup subtypes — highlight, underline, strikeout, squiggly — all work the same way. The rectangle bounds the annotation, but the actual marked region comes from setQuadPoints(): a flat array of eight numbers per quadrilateral, giving the corners of each marked text run.

use Papier\Annotation\HighlightAnnotation;

$hl = new HighlightAnnotation(72, 756, 400, 770);
$hl->setQuadPoints([72, 770, 400, 770, 72, 756, 400, 756])
   ->setColor(Color::rgb(1.0, 1.0, 0.0))
   ->setContents('Highlighted section');

$page->addAnnotation($hl);

The eight numbers are the upper-left, upper-right, lower-left and lower-right corners in that order. Note that this is not the order you would use to trace the outline — it is the order the specification requires.

Text spanning several lines needs one quadrilateral per line, concatenated into the same array:

$hl->setQuadPoints([
    72, 770, 400, 770, 72, 756, 400, 756,   // first line
    72, 754, 310, 754, 72, 740, 310, 740,   // second line, shorter
]);

Underline, strikeout and squiggly are drop-in replacements:

use Papier\Annotation\{StrikeOutAnnotation, SquigglyAnnotation, UnderlineAnnotation};

$ul = new UnderlineAnnotation(72, 740, 300, 754);
$ul->setQuadPoints([72, 754, 300, 754, 72, 740, 300, 740])
   ->setColor(Color::hex('#0000ff'));

$so = new StrikeOutAnnotation(72, 724, 200, 738);
$so->setQuadPoints([72, 738, 200, 738, 72, 724, 200, 724])
   ->setColor(Color::hex('#ff0000'))
   ->setContents('Delete this clause');

$sq = new SquigglyAnnotation(200, 724, 400, 738);
$sq->setQuadPoints([200, 738, 400, 738, 200, 724, 400, 724])
   ->setColor(Color::cmyk(0, 0.5, 1.0, 0));

Markup annotations do not read the page's text — they mark coordinates. If you are annotating text you laid out yourself, derive the quad points from the same positions you passed to Text::write(). For text you did not lay out, extract positions first (see Reading PDFs).

Geometric markup

Square and circle annotations draw a shape inscribed in the annotation rectangle. Both support a separate interior colour, so the border and the fill can differ:

use Papier\Annotation\{CircleAnnotation, SquareAnnotation};

$sq = new SquareAnnotation(320, 600, 500, 630);
$sq->setContents('Check these figures')
   ->setColor(Color::rgb(0.0, 0.5, 0.0))          // border
   ->setInteriorColor(Color::hex('#e5ffe5'))      // fill
   ->setBorderStyle(1.5);

$ci = new CircleAnnotation(72, 560, 200, 600);
$ci->setColor(Color::hex('#0000cc'))
   ->setInteriorColor(Color::hex('#e5e5ff'))
   ->setBorderStyle(1.5);

A circle annotation draws the ellipse inscribed in its rectangle, so a square rectangle gives a true circle and a wide one gives an ellipse.

LineAnnotation is unusual: it takes eight constructor arguments — the bounding rectangle followed by the line's two endpoints — and it requires an explicit finalize() call to write the endpoint array:

use Papier\Annotation\LineAnnotation;

//                        ---- rect ----  -- line endpoints --
$line = new LineAnnotation(72, 600, 300, 630, 72, 600, 300, 620);
$line->setContents('A line annotation with arrows')
     ->setColor(Color::rgb(0.8, 0.0, 0.0))
     ->setLineEndings('OpenArrow', 'ClosedArrow')
     ->setInteriorColor(Color::rgb(1.0, 0.8, 0.0))   // fills closed arrowheads
     ->setBorderStyle(2.0)
     ->finalize();                                    // required

Without finalize() the /L array is never written and the annotation is invalid. Call it last, after the other setters.

Line ending styles include None, Square, Circle, Diamond, OpenArrow, ClosedArrow, Butt, ROpenArrow, RClosedArrow and Slash. The interior colour fills the closed shapes.

PolygonAnnotation takes a flat list of alternating x and y coordinates and closes the shape automatically:

use Papier\Annotation\PolygonAnnotation;

$poly = new PolygonAnnotation(220, 555, 380, 605);
$poly->setVertices([220, 555, 300, 605, 380, 555, 340, 575, 260, 575])
     ->setContents('A polygon annotation')
     ->setColor(Color::cmyk(0, 1.0, 0, 0.5));

Make sure the bounding rectangle actually contains all the vertices — viewers clip to /Rect.

Stamps and ink

A stamp annotation shows a predefined graphic. setIcon() selects it from the standard set: Approved, Experimental, NotApproved, AsIs, Expired, NotForPublicRelease, Confidential, Final, Sold, Departmental, ForComment, TopSecret, Draft, ForPublicRelease.

use Papier\Annotation\StampAnnotation;

$stamp = new StampAnnotation(390, 555, 520, 605);
$stamp->setIcon('Draft')
      ->setContents('DRAFT')
      ->setColor(Color::hex('#ff0000'));

Viewers render the standard icons themselves, and their appearance varies. For a stamp that looks identical everywhere, supply your own appearance stream with setAppearance(), or draw the graphic on the page and use the stamp only to carry the metadata.

InkAnnotation captures freehand strokes. setInkList() takes an array of arrays: each inner array is one continuous stroke, given as alternating x and y coordinates. Separate strokes mean the pen was lifted.

use Papier\Annotation\InkAnnotation;

$ink = new InkAnnotation(72, 500, 300, 545);
$ink->setInkList([
    [72, 520, 100, 540, 140, 510, 180, 535, 220, 515, 260, 530, 300, 520],  // stroke 1
    [90, 505, 130, 505],                                                     // stroke 2
])
    ->setColor(Color::rgb(0.0, 0.6, 0.0))
    ->setBorderStyle(2.0)
    ->setContents('A freehand ink annotation');

The border style width sets the pen thickness.

Redaction

A redaction annotation marks content for removal. It does not remove anything by itself — that is a deliberate two-phase design, so a reviewer can mark regions, have them checked, and only then apply the redaction irreversibly.

use Papier\Annotation\RedactAnnotation;

$redact = new RedactAnnotation(72, 452, 300, 470);
$redact->setQuadPoints([72, 452, 300, 452, 72, 470, 300, 470])
       ->setOverlayText('REDACTED')
       ->setRepeat(true)                    // tile the overlay text across the region
       ->setFillColor(Color::black())       // the bar painted after application
       ->setContents('Sensitive content marked for redaction');

As with text markup, setQuadPoints() takes runs of eight numbers describing each quadrilateral. setFillColor() is the colour of the bar painted once a redaction tool applies the mark, and setOverlayText() is the label drawn over it.

A redaction annotation in a saved PDF is a marking only. The underlying text and images are still in the file and can be extracted. Do not distribute a document whose redactions have not been applied by a tool that actually strips the content.

Reading annotations

Annotations in an existing PDF can be listed with the parser:

use Papier\Parser\PdfParser;

$parser = new PdfParser(file_get_contents('reviewed.pdf'));
$parser->parse();

foreach ($parser->getAnnotations() as $annot) {
    print_r($annot);
}

See Reading PDFs for the full parsing API.
