Installation

Papier is distributed on Packagist and installs with a single Composer command. It has no runtime dependencies beyond three standard PHP extensions.

Requirements

RequirementVersionUsed for
PHP8.1 or newerEnums, readonly properties, and first-class callable syntax throughout the API
ext-zlibbundledFlateDecode stream compression, object streams, cross-reference streams
ext-mbstringbundledMulti-byte text encoding, Unicode fonts, /ToUnicode CMaps
ext-opensslbundledRC4/AES encryption, PKCS#7 digital signatures, document timestamps

All three extensions ship with most PHP distributions. Confirm they are enabled before installing:

php -r "printf('PHP %s%s', PHP_VERSION, PHP_EOL);"
php -m | grep -E '^(zlib|mbstring|openssl)$'

The second command should print all three names. If one is missing, install it through your platform's package manager (for example apt install php8.1-mbstring) or enable it in php.ini.

Optional: a few examples use the gd extension to synthesise sample images at runtime. It is not required by the library itself — image embedding works from file paths and raw bytes without it.

Install with Composer

composer require papier/papier

Then include Composer's autoloader in your script. Papier follows PSR-4 under the Papier\ namespace, so every class is autoloaded on first use.

require_once __DIR__ . '/vendor/autoload.php';

use Papier\PdfDocument;

Version constraints

Version 3.0 changed four public method signatures, replacing string parameters with enums. If you are pinning a constraint by hand, use a caret range so you receive patch and minor updates but not the next major:

composer require papier/papier:^3.0

Upgrading from 2.x? The changelog lists every breaking change with a before/after snippet. There are four call sites to update, and the output format is unchanged — regenerating a document after the upgrade produces identical bytes.

Verify your installation

Create test.php in your project root:

require_once __DIR__ . '/vendor/autoload.php';

use Papier\PdfDocument;
use Papier\Elements\{Color, Text};

$doc  = PdfDocument::create();
$font = $doc->addFont('Helvetica-Bold');
$page = $doc->addPage();

$page->add(
    Text::write('Papier is installed.')
        ->at(72, 720)
        ->font($font, 20)
        ->color(Color::hex('#2c3e7a')),
);

$doc->save(__DIR__ . '/test.pdf');

echo "Wrote test.pdf\n";

Run it and open the result:

php test.php

If test.pdf opens in your viewer with the line of text on an A4 page, the installation is complete. Continue with the quick start.

Installing from source

To work on the library itself, or to run the bundled examples and test suite, clone the repository and install the development dependencies:

git clone https://github.com/papier/papier.git
cd papier
composer install

This pulls in PHPUnit and PHPStan alongside the library.

Running the tests

vendor/bin/phpunit
vendor/bin/phpstan analyse

Running the examples

The examples/ directory holds 34 runnable scripts. Each writes its output to examples/output/:

php examples/01_hello_world.php

See Examples for the full catalogue.

Verifying rendered output

The repository includes a verification harness that renders every example with Ghostscript and runs qpdf --check against the result, confirming the files not only parse but actually display:

php tools/verify.php

This requires gs and qpdf on your PATH. It is a development tool — nothing in the library depends on either binary.

Framework integration

Papier has no framework coupling. In a Laravel or Symfony application, autoloading is already configured by Composer, so the classes are available immediately. To send a generated document as an HTTP response rather than writing it to disk, use output(), which emits the appropriate headers followed by the file:

$doc->output('report.pdf');          // inline, in the browser's viewer
$doc->output('report.pdf', false);   // as a download

When you need the bytes themselves — to return them from a controller, store them in S3, or pass them to a signer — use toString():

$pdf = $doc->toString();   // the complete file as a binary string

return response($pdf, 200, [
    'Content-Type'        => 'application/pdf',
    'Content-Disposition' => 'inline; filename="report.pdf"',
]);

Make sure no output — whitespace outside PHP tags, a debug echo, a BOM in an included file — is emitted before the headers and the PDF bytes. Any stray byte corrupts the file, and most viewers will refuse to open it with a misleading error.

Troubleshooting

Memory limits

Large documents, embedded fonts and high-resolution images all live in memory until the file is written. If you hit PHP's memory_limit on a big job, raise it for the script, or reduce the footprint with object streams and font subsetting, which cut the output size considerably.

Embedded font file not found

addFont() accepts either one of the 14 standard font names or a filesystem path to a .ttf or .otf file. A path that does not resolve throws rather than silently falling back, so check it with an absolute path built from __DIR__ when a relative path fails.

OpenSSL signature errors

Digital signing needs a certificate and a private key in PEM form. If PdfSigner reports that it cannot read them, verify the files independently with openssl x509 -in cert.pem -text -noout. See Encryption & signatures.
