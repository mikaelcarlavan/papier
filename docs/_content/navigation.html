Bookmarks & navigation

A long PDF is only usable if the reader can move around it. This page covers the machinery for that: the outline tree that fills the bookmarks panel, destinations that name places in the document, actions that respond to clicks, page labels that fix the numbering shown in the viewer, and transitions for presentation mode.

Outlines (bookmarks)

The document outline is the collapsible tree in the viewer's sidebar. It is built from two classes: PdfOutline is the root container, and PdfOutlineItem is a single bookmark that may have children.

use Papier\Structure\{PdfOutline, PdfOutlineItem};

$outline = new PdfOutline();

$intro = new PdfOutlineItem('Introduction');
$outline->addItem($intro);

$ch1 = new PdfOutlineItem('Chapter 1: Basics');
$outline->addItem($ch1);

$doc->setOutline($outline);

PdfOutline::addItem() takes a PdfOutlineItem object. Construct the item first with its title, configure it, then add it.

Nesting bookmarks

Hierarchy comes from addChild() on the item, not from the root. Only top-level items go through addItem():

$ch1 = new PdfOutlineItem('Chapter 1: Basics');
$ch1->addChild(new PdfOutlineItem('1.1 File structure'))
    ->addChild(new PdfOutlineItem('1.2 Objects'));

$sec13 = new PdfOutlineItem('1.3 Streams');
$sec13->addChild(new PdfOutlineItem('1.3.1 Filters'));
$ch1->addChild($sec13);

$outline->addItem($ch1);

Nesting is unlimited. Papier computes the /First, /Last, /Next, /Prev, /Parent and /Count links that the outline tree requires.

Styling bookmarks

Bookmarks can carry a colour and a bold or italic style — useful for distinguishing chapter headings from sections at a glance:

$ch1->setBold(true)
    ->setItalic(false)
    ->setColor(0.2, 0.3, 0.6);   // RGB components in [0, 1]

setColor() also accepts a Color object:

use Papier\Elements\Color;

$ch1->setColor(Color::hex('#334d99'));

Bookmark targets

A bookmark with no destination is just a label. Give it a target with setDestination(), which accepts either a named destination string or an explicit destination array:

use Papier\Destination\XYZDestination;

// By name — resolved through the document's name tree
$ch1->setDestination('chapter1');

// Explicit: scroll page 2 so that (72, 750) is at the top-left, zoom unchanged
$ch1->setDestination(XYZDestination::create($page2->getDictionary(), 72, 750, null));

Named destinations are usually the better choice: the bookmark refers to a name, the name refers to a page, and reorganising the document only means updating the name definition. Alternatively, setAction() attaches any action instead of a destination — for example a bookmark that opens a web page.

Destinations

A destination says which page to show and how to fit it in the window. Papier provides one class per fit style in Papier\Destination; each has a static create() returning the destination array.

ClassArguments after the pageBehaviour
XYZDestination$left, $top, $zoomPosition a specific point at the top-left, with an explicit zoom
FitDestination—Fit the whole page in the window
FitHDestination$topFit the page width, positioned at a given vertical coordinate
FitVDestination$leftFit the page height, positioned at a given horizontal coordinate
FitRDestination$left, $bottom, $right, $topFit a specific rectangle in the window
FitBDestination—Fit the page's bounding box (its inked area)
FitBHDestination$topFit the bounding box width
FitBVDestination$leftFit the bounding box height
 
use Papier\Destination\{FitDestination, FitHDestination, FitRDestination};

$whole  = FitDestination::create($page->getDictionary());
$byWide = FitHDestination::create($page->getDictionary(), 750);
$onFig  = FitRDestination::create($page->getDictionary(), 72, 400, 523, 640);

In XYZDestination, passing null for a parameter means "leave this as the reader had it" — so XYZDestination::create($p, null, 750, null) scrolls vertically without changing the horizontal position or the zoom. Passing 0 for the zoom also means "unchanged", which is why null is the clearer way to express it.

FitDestination is the friendliest default for bookmarks: it makes no assumption about the reader's window size or preferred zoom. Fixed XYZ zooms can be jarring on small screens.

Named destinations

A named destination gives a stable, human-readable label to a place in the document. Links, bookmarks and external references can then target the name rather than a page object, and the target survives editing.

$doc->addNamedDestination('intro',    $page1, 72, 750);
$doc->addNamedDestination('chapter1', $page2, 72, 780);
$doc->addNamedDestination('appendix', $page9, null, null, null);

The signature is addNamedDestination(string $name, PdfPage $page, ?float $left, ?float $top, ?float $zoom). It always creates an XYZ destination; omit or pass null for the coordinates you want to leave alone.

Named destinations are also what makes cross-document linking work — GoToRAction can open another PDF at a named destination without knowing its page numbering.

The open action

By default a viewer opens a document at page one. setOpenDestination() overrides that:

$doc->setOpenDestination($page, 0, 841);   // top-left of the page
$doc->setOpenDestination($tocPage);        // the table of contents, position unchanged

The parameters mirror addNamedDestination(): a page plus optional left, top and zoom. This is how you make a report open on its summary, or a manual open on its index.

Related presentation settings — whether the bookmarks panel is open on load, whether the toolbar is hidden — live in viewer preferences, covered in Documents & pages.

Actions

An action is something that happens when the user activates a link, a bookmark, or a form field. Papier's Papier\Action namespace has a class per action type, each extending the abstract Action.

Navigation actions

use Papier\Action\{GoToAction, GoToRAction, NamedAction, URIAction};

// Jump within this document — by name or by explicit destination
$goto = new GoToAction('chapter1');
$goto = new GoToAction(XYZDestination::create($page->getDictionary(), 72, 750, null));

// Open a URL in the browser
$uri = new URIAction('https://example.com');

// Open a destination in another PDF file
$remote = new GoToRAction(/* file spec, destination, new window */);

// Trigger a built-in viewer command
$next = new NamedAction('NextPage');   // also FirstPage, LastPage, PrevPage, Print

URIAction takes an optional second argument, $isMap, which appends the click coordinates to the URL for server-side image maps.

Interactive actions

use Papier\Action\{HideAction, JavaScriptAction, LaunchAction, ResetFormAction, SubmitFormAction};

// Run a script (see the warning below)
$js = new JavaScriptAction('app.alert("Please complete all required fields.");');

// Reset every form field to its default value
$reset = new ResetFormAction();

// Show or hide annotations, addressed by their /NM name
$hide = new HideAction($annotationRef, true);

// Launch an external application or file
$launch = new LaunchAction($fileSpec, newWindow: true);

SubmitFormAction posts form data to a URL, and pairs naturally with a push button — see Forms.

JavaScript actions are disabled by default in most modern viewers and are forbidden outright in PDF/A. Never make required functionality depend on one — see Tagged PDF & PDF/A.

Feature actions

The remaining actions drive specific document features: SetOCGStateAction toggles optional content layers, RenditionAction controls media playback, SoundAction plays an embedded sound, MovieAction drives a legacy movie annotation, TransitionAction triggers a page transition, ThreadAction follows an article thread, ImportDataAction loads form data from a file, GoToEAction targets an embedded file, and GoTo3DViewAction selects a view in a 3D annotation.

use Papier\Action\SetOCGStateAction;

$toggle = new SetOCGStateAction($stateArray);   // see the multimedia page for state arrays

Chaining actions

setNext() makes one action run after another, so a single click can do several things:

$jump = new GoToAction('summary');
$jump->setNext($someOtherAction->getDictionary());

Link annotations

Actions and destinations need something to activate them. That is the job of the link annotation — an invisible rectangle that responds to clicks.

use Papier\Annotation\LinkAnnotation;

$link = new LinkAnnotation(72, 680, 272, 700);
$link->setURI('https://example.com')
     ->setBorderStyle(0);        // suppress the default blue rectangle
$page->addAnnotation($link);

setURI() is a shortcut for the common web-link case. For anything else, attach an action or a destination:

use Papier\Action\GoToAction;
use Papier\Destination\FitDestination;

// Internal jump via an action
$internal = new LinkAnnotation(72, 650, 272, 670);
$internal->setAction(new GoToAction('chapter1'))
         ->setBorderStyle(0);

// Internal jump via a destination array
$direct = new LinkAnnotation(72, 620, 272, 640);
$direct->setDestination(FitDestination::create($page5->getDictionary()))
       ->setBorderStyle(0);

setHighlightMode() controls the visual feedback while the mouse is down: N none, I invert, O outline, P push.

A link is a hotspot with no appearance of its own, so draw the visible affordance yourself:

use Papier\Elements\{Color, Text};

$page->add(
    Text::write('See Chapter 1')->at(80, 686)->font($font, 11)->color(Color::rgb(0.1, 0.3, 0.8)),
);
$page->addAnnotation($internal);

Full annotation coverage is in Annotations.

Page labels

Page labels decouple the numbering the reader sees from the physical page index. Front matter numbered i, ii, iii followed by a body starting at 1 is the classic case: without labels the viewer's page box and the printed folio disagree, and jumping to "page 12" lands in the wrong place.

use Papier\Structure\PageLabelStyle;

$doc->addPageLabel(0, PageLabelStyle::RomanLower);   // pages 1-4 shown as i, ii, iii, iv
$doc->addPageLabel(4, PageLabelStyle::Decimal);      // page 5 onward shown as 1, 2, 3 …

Each call opens a range starting at a 0-based page index and running until the next range begins. The styles are:

StyleProduces
PageLabelStyle::Decimal1, 2, 3 …
PageLabelStyle::RomanLoweri, ii, iii …
PageLabelStyle::RomanUpperI, II, III …
PageLabelStyle::AlphaLowera, b, c … aa, bb …
PageLabelStyle::AlphaUpperA, B, C … AA, BB …
PageLabelStyle::NoneNo number — prefix only

Two further parameters control the range. $startValue sets the first number, and $prefix is prepended to every label:

$doc->addPageLabel(4, PageLabelStyle::Decimal, prefix: 'p.');        // p.1, p.2, p.3 …
$doc->addPageLabel(12, PageLabelStyle::Decimal, startValue: 100);   // 100, 101, 102 …
$doc->addPageLabel(20, PageLabelStyle::None, prefix: 'Appendix');   // Appendix, Appendix …

The full signature is addPageLabel(int $startPage, PageLabelStyle $style, int $startValue, string $prefix).

Page transitions

Transitions animate the change from one page to the next in full-screen presentation mode. They are ignored during normal scrolling, so they cost nothing in an ordinary document.

use Papier\Structure\PageTransition;

$page->setTransition(new PageTransition(PageTransition::DISSOLVE, 1.0));

The constructor takes a style constant and a duration in seconds. The available styles:

ConstantEffect
REPLACEInstant swap — the default, no animation
SPLITTwo lines sweep inward or outward
BLINDSMultiple parallel lines sweep across
BOXA rectangle expands or contracts from the centre
WIPEA single line sweeps across the page
DISSOLVERandom pixel dissolve
GLITTERDissolve with a directional sweep
FLYThe new page flies in (PDF 1.5)
PUSHThe new page pushes the old one off (PDF 1.5)
COVERThe new page slides over the old (PDF 1.5)
UNCOVERThe old page slides away (PDF 1.5)
FADECross-fade through the backdrop (PDF 1.5)

Several styles take a direction or motion modifier:

// Wipe left to right
$page->setTransition(
    (new PageTransition(PageTransition::WIPE, 0.6))->setDirection(0)
);

// Box expanding outward
$page->setTransition(
    (new PageTransition(PageTransition::BOX, 0.8))->setMotion(PageTransition::MOTION_OUT)
);

// Horizontal blinds
$page->setTransition(
    (new PageTransition(PageTransition::BLINDS, 0.7))->setDimension(PageTransition::DIM_H)
);

setDirection() takes degrees — 0 left-to-right, 90 bottom-to-top, 180 right-to-left, 270 top-to-bottom, and 315 for the diagonal used by GLITTER. setMotion() takes MOTION_IN or MOTION_OUT. setDimension() takes DIM_H or DIM_V. For FLY, setStartingScale() and setOpaque() tune how the incoming page enters.

Auto-advance

Pair a transition with setDuration() on the page to build a self-running slideshow. Note the naming: the duration passed to PageTransition is how long the animation takes, while PdfPage::setDuration() is how long the page is displayed before advancing.

$page1->setTransition(new PageTransition(PageTransition::DISSOLVE, 1.0));
$page1->setDuration(5.0);   // hold for 5 s, then advance automatically

To make the document open straight into full-screen mode, set the page mode through viewer preferences — see Documents & pages.
