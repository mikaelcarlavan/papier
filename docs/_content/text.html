Text & fonts

Papier gives you two levels of text API: the Text and TextBox elements for ordinary typesetting, and ContentStream for the PDF text operators underneath. Fonts are registered once on the document and referenced everywhere by the resource name that registration returns.

Registering a font

Before you can draw any text you must register a font with the document. addFont() returns a short resource name — F1, F2 and so on — that identifies the font inside content streams. You pass that string to every element that draws text.

use Papier\PdfDocument;
use Papier\Elements\Text;

$doc  = PdfDocument::create();
$page = $doc->addPage();

$regular = $doc->addFont('Helvetica');
$bold    = $doc->addFont('Helvetica-Bold');

$page->add(Text::write('Hello')->at(72, 750)->font($bold, 24));

The variables $regular and $bold hold strings, not font objects. Registering the same font twice returns the same name, so it is safe to call addFont() wherever it reads best. You can override the generated name with the second argument when you need a predictable key inside a hand-written content stream.

$doc->addFont('Helvetica-Bold', 'FBig');   // always resource /FBig

The 14 standard fonts

Every conforming PDF reader ships with fourteen Type 1 fonts, so documents using them need embed nothing at all. This keeps files tiny and is the right default for internal or short-lived documents.

$doc->addFont('Helvetica');
$doc->addFont('Times-Roman');
$doc->addFont('Courier');

The names are case-sensitive and must be spelled exactly as above. Papier ships the AFM metrics for all fourteen, so line-breaking and table column measurement are accurate rather than estimated.

Standard fonts are not embedded, which means the reader substitutes its own copy. That is fine for screen viewing but is forbidden by PDF/A. Archival documents must embed every font — see embedded fonts below.

The Text element

Text draws a single run of text at an exact position. It does not wrap, does not measure, and does not interpret newlines — it is the direct equivalent of positioning the text cursor and showing a string.

use Papier\Elements\{Color, Text};

$page->add(
    Text::write('Quarterly Report')
        ->at(72, 780)
        ->font($bold, 20)
        ->color(Color::hex('#1a1a2e')),
);

The anchor passed to at() is the baseline origin of the first glyph, not the top-left of a box. Descenders on letters like g and y extend below it.

MethodPurpose
Text::write(string $text)Create the element
at(float $x, float $y)Baseline origin in points
font(string $fontName, float $size)Resource name from addFont(), size in points
color(Color $color)Fill colour
rgb(float $r, float $g, float $b)Shorthand for an RGB fill colour
opacity(float $opacity)Fill alpha from 0 to 1

All the setters return the element, so they chain in any order. opacity() is the natural way to build a watermark:

$page->add(
    Text::write('DRAFT')->at(150, 400)->font($bold, 90)->opacity(0.12),
);

Colours

Color is a small value object with four constructors covering the device colour spaces, plus two named shorthands. All component values run from 0 to 1.

use Papier\Elements\Color;

Color::rgb(0.2, 0.5, 0.9);
Color::hex('#e74c3c');            // also accepts 'e74c3c' and 3-digit forms
Color::gray(0.35);
Color::cmyk(0.0, 0.8, 0.7, 0.1);  // for print output
Color::black();
Color::white();

Choose CMYK when the document is destined for a commercial press and the colours must be specified in ink rather than light; otherwise RGB is both smaller to encode and what screen readers expect.

The TextBox element

TextBox lays a paragraph out inside a bounding width, breaking lines at word boundaries. Use it whenever the text is long enough that you cannot predict where it ends — which is to say, whenever the content is data rather than a label.

use Papier\Elements\TextBox;

$page->add(
    TextBox::write(
        'The TextBox element automatically wraps long text to fit a bounding '
      . 'width, using font metrics for accurate line breaking. It supports '
      . 'left, center, and right alignment and a configurable line height.'
    )
        ->at(72, 700)
        ->size(451, 80)
        ->font($regular, 11, 'Times-Roman')
        ->lineHeight(1.5)
        ->align('left'),
);
MethodPurpose
TextBox::write(string $text)Create the element
at(float $x, float $y)Top-left corner of the box
size(float $width, float $height = 0)Wrap width; height 0 means unbounded
font(string $fontName, float $size, string $baseFontName = '')Font, size, and optional metrics hint
lineHeight(float $multiplier)Line spacing as a multiple of font size
align(string $align)'left', 'center' or 'right'
color(Color $color) / rgb(…)Text colour

The third argument to font() deserves explanation. To break lines correctly the box must know how wide each character is. When you pass the base font name — 'Times-Roman', 'Helvetica' — Papier looks up the real AFM metrics. Omit it and it falls back to a crude estimate of 0.55 times the font size per character, which produces visibly ragged or overflowing lines.

Pass the base font name whenever you use a standard font. For embedded TrueType fonts the metrics come from the font file itself and the hint is unnecessary.

A height of 0 lets the paragraph run as long as it needs. Giving a real height clips the text to the box, which is what you want in fixed-layout designs where overflow must not collide with the next block. See examples/02_text_formatting.php.

Embedded TrueType and OpenType fonts

Pass a file path to addFont() and Papier parses the font binary, extracts the PostScript name from its name table, reads metrics from hhea, OS/2 and hmtx, and embeds the font program as a FontFile2 stream.

$lato     = $doc->addFont(__DIR__ . '/fonts/Lato-Regular.ttf');
$latoBold = $doc->addFont(__DIR__ . '/fonts/Lato-Bold.ttf');

$page->add(
    Text::write('Lato at 18 pt')->at(72, 680)->font($lato, 18),
);

Embedding guarantees the document renders identically everywhere, and is mandatory for PDF/A. The cost is file size: a full Latin TrueType face adds a few hundred kilobytes, which is where subsetting comes in.

These fonts are encoded as single-byte WinAnsi, giving you 256 code points. That covers Western European Latin — accented characters such as é, à, ü and ñ work without any special handling — but not Greek, Cyrillic or CJK. For those, see Unicode fonts. Papier also implements MacRoman encoding for documents that need it.

Subsetting

Subsetting strips every glyph outline that your document does not actually use, then rewrites the font tables to match. A report using two hundred distinct characters ends up embedding two hundred glyphs rather than several thousand.

$lato = $doc->addFont(__DIR__ . '/fonts/Lato-Regular.ttf', subset: true);

Subsetting is off by default for addFont() because it requires Papier to know the complete set of characters in advance, which it collects as you add elements. If you build content in unusual ways — writing raw operators that Papier cannot inspect — a glyph could be dropped. In the normal element-based flow this does not happen.

ToUnicode and searchable text

A subset font's internal glyph codes bear no relation to Unicode, so without extra information a reader has no idea that glyph 37 means the letter "e". Text would render correctly but be unsearchable and would copy out as gibberish.

Papier avoids this by generating a /ToUnicode CMap automatically, mapping every code back to its Unicode value. Search, copy-paste, screen readers and text extraction all keep working. You do not need to do anything to enable it.

See examples/20_font_subsetting.php for a before-and-after size comparison, and examples/29_text_extraction.php for extraction round-tripping through accented and subset text.

Full Unicode and CJK — Type 0 fonts

When 256 code points are not enough, use addUnicodeFont(). It embeds the font as a CIDFontType2 descendant of a Type 0 composite font with Identity-H encoding, which addresses glyphs with two-byte codes and so reaches every glyph in the file.

$noto = $doc->addUnicodeFont(__DIR__ . '/fonts/NotoSansCJK-Regular.otf');

$page->add(
    Text::write('日本語 / 中文 / 한국어 / Ελληνικά / Русский')
        ->at(72, 700)
        ->font($noto, 16),
);

The returned resource name is used exactly like any other, so switching a document from addFont() to addUnicodeFont() is a one-line change. Papier handles the encoding of your UTF-8 strings into two-byte glyph codes internally.

Subsetting is on by default here, and for good reason: a full CJK font can exceed 15 MB. A /ToUnicode CMap is generated as always, so the text stays searchable.

$doc->addUnicodeFont($path);                    // subset (default)
$doc->addUnicodeFont($path, subset: false);     // embed every glyph
$doc->addUnicodeFont($path, 'FCJK');            // fixed resource name

Use addFont() when your text is Western European and file size or simplicity matters; use addUnicodeFont() whenever any other script appears. There is no penalty to using the Unicode path for Latin text beyond slightly larger encoded strings.

See examples/21_unicode_cjk.php.

Type 3 fonts

A Type 3 font defines each glyph as a content stream of arbitrary graphics operators rather than an outline in a font format. That makes it the tool for icon sets, custom symbols and bitmap glyphs — anything you want to position, size and colour as if it were text.

use Papier\Content\ContentStream;
use Papier\Font\Type3Font;

$icons = new Type3Font();
$icons->setFontBBox(0, 0, 1000, 1000);

$disc = (new ContentStream())->drawCircle(500, 350, 350)->fill();
$icons->addGlyph(65, $disc, 1000, 'disc');          // code 65 = 'A'

$square = (new ContentStream())->rectangle(150, 0, 700, 700)->fill();
$icons->addGlyph(66, $square, 1000, 'square');      // 'B'

$tri = (new ContentStream())->moveTo(500, 700)->lineTo(150, 0)->lineTo(850, 0)->closePath()->fill();
$icons->addGlyph(67, $tri, 1000, 'triangle');       // 'C'

$iconFont = $doc->registerFont($icons, 'Icons');

$page->add(Text::write('ABC')->at(72, 680)->font($iconFont, 48)->color(Color::hex('#c0392b')));

The arguments to addGlyph() are the character code, the drawing stream, the advance width, and an optional glyph name. Glyph coordinates live in a 1000-unit design space by convention, mapped into text space by the font matrix; setFontMatrix() lets you change that mapping if you prefer a different scale.

Because the glyphs are content streams, they inherit the current fill colour — which is why 'ABC' above comes out red. Note that a Type 3 font is not registered with addFont() but with registerFont(), which takes a Font object. See examples/26_type3_font.php.

Typography via ContentStream

The element API covers position, font, size and colour. Everything else in the PDF text model — spacing, scaling, rise, rendering modes — is reached through ContentStream, which you build yourself and hand to PdfPage::addContent().

Text operators only apply between beginText() and endText(). Inside that block you set state, position the cursor, and show strings.

use Papier\Content\ContentStream;

$cs = new ContentStream();
$cs->beginText()
   ->setFont($regular, 12)
   ->setTextPosition(72, 600)
   ->showText('Normal text')
   ->endText();

$page->addContent($cs);

Character and word spacing

setCharSpacing() adds a fixed amount to every glyph advance and setWordSpacing() adds to the advance of the space character only. Both take points and both accept negatives for tightening.

$cs->beginText()
   ->setFont($regular, 12)
   ->setTextPosition(72, 580)->setCharSpacing(3)
   ->showText('L e t t e r s p a c e d')
   ->setTextPosition(72, 560)->setCharSpacing(0)->setWordSpacing(10)
   ->showText('Wide word spacing here')
   ->endText();

Letterspacing is conventional for small-caps headings and all-caps labels. Word spacing is mostly useful for crude justification, though the reader will not rebreak lines for you.

Both settings are part of the graphics state and persist until changed. Reset them to 0 before showing normal text, or wrap the styled run in save() and restore().

Horizontal scaling, rise and leading

$cs->beginText()
   ->setFont($regular, 12)
   ->setHorizontalScaling(80)     // condense to 80% width
   ->setTextRise(4)               // raise baseline: superscript
   ->setTextLeading(18)           // line spacing for nextLine()
   ->setTextPosition(72, 520)
   ->showText('First line')
   ->nextLine()
   ->showText('Second line, 18 pt below')
   ->endText();

setHorizontalScaling() takes a percentage; 100 is normal. Stretching glyphs is a poor substitute for a properly condensed font face, but it is occasionally the only way to squeeze a fixed string into a fixed width. setTextRise() shifts the baseline and, paired with a smaller size, produces superscripts and subscripts. setTextLeading() defines the vertical step that nextLine() takes.

Text rendering modes

The rendering mode decides whether glyphs are filled, stroked, both, invisible, or added to the clipping path. Outlined headings and invisible OCR layers both come from this one setting.

$cs->beginText()
   ->setFont($bold, 32)
   ->setFillRGB(1, 1, 1)
   ->setStrokeRGB(0.1, 0.2, 0.6)
   ->setLineWidth(0.7)
   ->setTextRenderMode(1)          // stroke only: outlined letters
   ->setTextPosition(72, 460)
   ->showText('OUTLINED')
   ->setTextRenderMode(0)          // always reset
   ->endText();
ModeEffect
0Fill (the default)
1Stroke
2Fill then stroke
3Invisible — renders nothing but remains searchable
4Fill and add to clipping path
5Stroke and add to clipping path
6Fill, stroke and add to clipping path
7Add to clipping path only

Mode 3 is how scanned documents carry an invisible OCR text layer over the page image: the words are selectable and searchable, but only the scan is visible. Modes 4 to 7 add the glyph outlines to the clip path, so subsequent painting — an image, a gradient — shows through the letterforms. See Graphics for clipping, and examples/27_shading_patterns.php for filling text with a gradient pattern.

Positioning and manual kerning

$cs->beginText()
   ->setFont($regular, 12)
   ->setTextPosition(72, 420)
   ->showTextArray(['A', -120, 'V', -80, 'ATAR'])
   ->endText();

showTextArray() interleaves strings with numeric adjustments expressed in thousandths of the font size, subtracted from the advance. Positive numbers pull glyphs closer, negative push them apart — note the sign is inverted relative to intuition. This is the mechanism readers use for kerning, and it is available if you are implementing your own kern-pair logic.

Other positioning helpers: moveText($tx, $ty) offsets relative to the start of the current line, moveTextSetLeading() does the same while setting the leading, setTextMatrix() installs an arbitrary transform for rotated or sheared text, and moveShowText() advances a line and shows a string in one operator.

A one-line shorthand

When you are already inside a hand-written stream and just need a label, writeText() wraps the whole begin/set/position/show/end sequence.

$cs->writeText('Figure 1', $regular, 9, 72, 400);

For anything more than a label, prefer the Text element — it is shorter still and participates in the element pipeline.
