Papier

A comprehensive PHP library for generating and reading PDF documents — implementing ISO 32000-1:2008 (PDF 1.7) with selected ISO 32000-2 (PDF 2.0) features.

Get started Browse examples GitHub

What is Papier?

Papier is a zero-dependency PHP library that writes and reads PDF files directly, rather than wrapping an external binary or a rendering engine. Everything happens in PHP: object serialisation, cross-reference construction, stream compression, font embedding, encryption, and parsing.

It covers the whole lifecycle of a document. You can compose a page with a fluent element API, drop down to raw PDF operators when you need exact control, then open the result back up, extract its text, fill its form fields, sign it, or append a revision without rewriting a byte of the original.

use Papier\PdfDocument;
use Papier\Elements\{Color, Text};

$doc  = PdfDocument::create();
$font = $doc->addFont('Helvetica-Bold');
$page = $doc->addPage();

$page->add(
    Text::write('Hello, World!')->at(72, 720)->font($font, 24)->color(Color::black()),
);

$doc->save('hello.pdf');

Highlights

Documents & pages

Metadata (Info + XMP), viewer preferences, page labels, named destinations, rotation, and page operations: merge, split, N-up, import.

Text & fonts

14 standard fonts, embedded TrueType/OpenType with subsetting and automatic /ToUnicode, Type 0 composite fonts for full Unicode and CJK, plus Type 3 glyph fonts.

Graphics

Paths, Bézier curves, clipping, nine colour spaces, four function types, axial/radial/mesh shadings, tiling and shading patterns, blend modes and soft masks.

Tables & layout

Column widths, header and footer rows, rowspan, per-side borders, padding, vertical alignment, and word-wrapped text boxes.

Forms & annotations

Seven AcroForm field types, 14 annotation subtypes including redaction, and a form filler that regenerates appearance streams on existing PDFs.

Security

RC4, AES-128 and AES-256 encryption with permission flags; PKCS#7 digital signatures with a visible appearance; RFC 3161 document timestamps.

Accessibility & archival

Tagged PDF with a full structure tree and /ParentTree, plus PDF/A-2b and accessible PDF/A-2a output with a built-in validator.

Reading & parsing

Layout- and ToUnicode-aware text extraction, object streams, hybrid-reference and linearized files, and xref recovery for damaged documents.

Why Papier?

Specification first

Papier is written against ISO 32000-1:2008 rather than against a set of common use cases. Structures that other libraries approximate — cross-reference streams, object streams, the structure tree, mesh shadings, CID font descriptors — are implemented as the specification defines them. That matters when your output has to survive a validator, an archival ingest pipeline, or another vendor's parser.

Read and write, not just write

Most PHP PDF libraries only generate. Papier also parses: it reads classic xref tables, cross-reference streams, object streams and hybrid-reference files, decrypts RC4/AES documents, and recovers files whose startxref is corrupt. The same object model backs both directions, so an incremental update or a signature can append to a file it has just read without disturbing the original bytes.

Two levels of API

The element API — Text, TextBox, Image, Table, Rectangle, Circle, Line — is fluent and immediate. Underneath it, ContentStream exposes the PDF operators themselves (re, m, l, c, Tf, Tj, …). You start high and drop down only where you need to, in the same document.

Type-safe since 3.0

Stringly-typed options were replaced by enums and fluent builders in version 3.0. Conformance levels, page rules, encryption algorithms and viewer preferences are now discoverable through IDE autocompletion, and an invalid value is a type error rather than a silently wrong PDF. See the changelog for the migration notes.

Requirements

  • PHP 8.1 or newer
  • ext-zlib — stream compression (FlateDecode)
  • ext-mbstring — multi-byte text handling
  • ext-openssl — encryption and digital signatures

No other runtime dependencies. See Installation for details.
