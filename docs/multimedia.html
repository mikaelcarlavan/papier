Layers & multimedia

Two features that make a PDF more than a printed page: optional content, which lets the reader switch parts of the page on and off, and multimedia, which embeds or references audio and video for playback inside the document.

Optional content (layers)

Optional content groups — OCGs, shown as "Layers" in most viewers — let a single page carry several alternative or supplementary renderings. A CAD drawing can hide its dimension annotations, a map can toggle its road and terrain layers, a multilingual form can show one language at a time. All the content is present in the file; the viewer simply decides what to paint.

Three pieces have to line up:

  1. An OCGroup for each layer.
  2. Each group registered in the page's resources under a property name, and the page content wrapped in marked-content operators that reference that name.
  3. An OCProperties object in the document catalog listing every group and the default visibility configuration.

Defining groups

An OCGroup is little more than the name the viewer shows in its layers panel:

use Papier\OptionalContent\OCGroup;

$bgGroup    = new OCGroup('Background');
$textGroup  = new OCGroup('Text Content');
$annotGroup = new OCGroup('Annotations');

setIntent() declares what the layer is for — 'View' (the default) means it affects display, 'Design' means it is metadata that viewers should not offer to toggle. setUsage() attaches a usage dictionary describing conditions such as zoom range or print behaviour.

Binding content to a layer

This is the step with no high-level shortcut. Layer membership is expressed with the BDC / EMC marked-content operators inside the content stream, so you need ContentStream directly rather than the elements API.

First register each group's dictionary in the page resources under a property name:

$page->getResources()->addProperties('OCBackground',  $bgGroup->getDictionary());
$page->getResources()->addProperties('OCText',        $textGroup->getDictionary());
$page->getResources()->addProperties('OCAnnotations', $annotGroup->getDictionary());

Then wrap the drawing operations in a marked-content sequence tagged OC and pointing at that property name:

use Papier\Content\ContentStream;

$cs = new ContentStream();

$cs->beginMarkedContentProps('OC', 'OCBackground');
$cs->setFillRGB(0.95, 0.95, 1.0)
   ->drawRect(0, 0, 595, 841, true, false);
$cs->setStrokeRGB(0.5, 0.5, 0.8)->setLineWidth(3)
   ->drawRect(30, 30, 535, 781, false, true);
$cs->endMarkedContent();

$cs->beginMarkedContentProps('OC', 'OCText');
$cs->beginText()
       ->setFont('F2', 26)->setFillRGB(0.2, 0.2, 0.6)
       ->setTextPosition(72, 760)
       ->showText('Optional Content Demo')
   ->endText();
$cs->endMarkedContent();

$page->addContent($cs);

Everything between beginMarkedContentProps() and the matching endMarkedContent() belongs to that layer and disappears when the reader switches it off. Content outside any marked-content sequence is always visible.

Every beginMarkedContentProps() must have a matching endMarkedContent(). Unbalanced markers produce a content stream that viewers reject or render unpredictably.

The related beginMarkedContent($tag) starts a sequence with no property list, and beginMarkedContentMcid($tag, $mcid) starts one carrying a marked-content identifier — that variant is for structural tagging rather than layers, and is covered in Tagged PDF & PDF/A.

The OCProperties entry

Finally, tell the document which groups exist and which are visible when it opens:

use Papier\OptionalContent\OCProperties;

$ocProps = new OCProperties();
$ocProps->addOCG($bgGroup)
        ->addOCG($textGroup)
        ->addOCG($annotGroup)
        ->setDefaultConfig(
            name:      'Default',
            on:        [$bgGroup, $textGroup],
            off:       [$annotGroup],
            baseState: 'ON',
        );

$doc->setOCProperties($ocProps);

Every group referenced anywhere in the document must be passed to addOCG() — a group that appears in a content stream but not in /OCProperties makes the file invalid.

setDefaultConfig() takes the configuration name shown in the viewer UI, arrays of groups that start on and off, and a $baseState of 'ON', 'OFF' or 'Unchanged' governing groups in neither list. Starting from baseState: 'OFF' and listing only what should be visible is convenient for documents with many layers where most are hidden by default.

Membership dictionaries

An OCMembership makes visibility depend on a combination of groups rather than a single one — content that appears only when two layers are both on, for instance.

use Papier\OptionalContent\OCMembership;

$both = new OCMembership([$layerA, $layerB], 'AllOn');
$page->getResources()->addProperties('OCBoth', $both->getDictionary());

The policy is 'AnyOn' (the default — visible if any listed group is on), 'AllOn', 'AnyOff', or 'AllOff'. Reference the membership from a marked-content sequence exactly as you would a plain group.

Toggling layers from an action

A button or link can change layer visibility using SetOCGStateAction, so the document itself can offer a "show annotations" control. See Bookmarks & navigation for how actions attach to links and form fields.

Multimedia

PDF gained media support in stages, and the three surviving mechanisms are worth distinguishing before choosing one.

MechanismIntroducedBest for
SoundAnnotationPDF 1.2Short audio embedded directly in the file
MovieAnnotationPDF 1.2Legacy video; superseded, kept for compatibility
ScreenAnnotation + renditionPDF 1.5Modern audio and video with MIME types and playback control

Media playback depends entirely on the viewer. Adobe Acrobat supports renditions; most browser-based viewers ignore them and show only the static appearance. Always draw a meaningful placeholder so the page still makes sense when playback is unavailable.

Embedded sound

SoundStream wraps raw audio samples as a PDF sound object. Static factories cover the encodings the format supports:

use Papier\Multimedia\SoundStream;

$sound = SoundStream::fromPcm($pcmBytes, 44100, 1, 16);  // data, rate, channels, bits
$sound = SoundStream::fromMuLaw($bytes, 8000, 1);        // G.711 μ-law, telephony
$sound = SoundStream::fromALaw($bytes, 8000, 1);         // G.711 A-law
$sound = SoundStream::fromFile('chime.wav');             // read from disk

PCM data is interleaved and little-endian. Generating a tone directly is a compact way to test playback:

$sampleRate = 44100;
$numSamples = (int) ($sampleRate * 0.5);
$pcm        = '';
for ($i = 0; $i < $numSamples; $i++) {
    $env    = min(1.0, min($i / 200.0, ($numSamples - $i) / 200.0));  // fade in/out
    $sample = (int) (28000 * $env * sin(2.0 * M_PI * 440.0 * $i / $sampleRate));
    $pcm   .= pack('v', $sample & 0xFFFF);
}
$sound = SoundStream::fromPcm($pcm, $sampleRate, 1, 16);

Attach it to the page with a SoundAnnotation, which renders as a clickable icon. Note that setSound() takes the underlying stream, obtained with getStream():

use Papier\Annotation\SoundAnnotation;
use Papier\Elements\Color;

$annot = new SoundAnnotation(72, 695, 92, 715);
$annot->setSound($sound->getStream())
      ->setIcon('Speaker')                  // 'Speaker' or 'Mic'
      ->setContents('440 Hz tone (0.5 s)')
      ->setColor(Color::rgb(0.1, 0.5, 0.8));

$page->addAnnotation($annot);

Embedded PCM is uncompressed — roughly 88 kB per second at CD quality. Keep embedded clips short, or reference a compressed file externally instead.

Media clips and file references

A MediaClip describes a piece of media: where it lives and what MIME type it is. The choice of factory decides whether the media travels with the PDF.

use Papier\Multimedia\MediaClip;

// External reference — the file must sit next to the PDF
$video = MediaClip::fromFile('demo.mp4', 'video/mp4', 'Demo Video');

// Embedded — the bytes are copied into the PDF, making it self-contained
$video = MediaClip::fromEmbedded('/path/to/demo.mp4', 'video/mp4', 'Demo Video');

// From a file specification you built yourself
$video = MediaClip::fromSpec($fileSpec, 'video/mp4', 'Demo Video');

External references keep the PDF small but break if the file moves. Embedding produces a self-contained document at the cost of size. FileSpec exposes the same choice directly:

use Papier\Multimedia\FileSpec;

$spec = FileSpec::external('demo.mp4', 'video/mp4');
$spec = FileSpec::embedded('/path/to/demo.mp4', 'video/mp4');
$spec = FileSpec::fromBytes('demo.mp4', $bytes, 'video/mp4');

setPermissions() on the clip attaches a MediaPermissions dictionary controlling whether the viewer may write a temporary file during playback — TEMP_ACCESS, NO_OPS, or NEVER_WRITTEN.

Play parameters

MediaPlayParams controls how the clip plays:

use Papier\Multimedia\MediaPlayParams;

$params = (new MediaPlayParams())
    ->setVolume(90)          // 0-100
    ->setShowControls(true)  // transport bar
    ->setAutoPlay(false)     // play on activation rather than on page open
    ->setRepeatCount(1.0)    // 0.0 = loop forever
    ->setFitStyle(MediaPlayParams::FIT_MEET);

The fit styles govern how the media is scaled into the annotation rectangle:

ConstantBehaviour
FIT_MEETScale proportionally so the whole clip fits — letterboxed
FIT_SLICEScale proportionally to fill the box — cropped
FIT_FILLStretch to fill, ignoring aspect ratio
FIT_SCROLLShow at natural size with scroll bars
FIT_HIDDENDo not display the media
FIT_TIGHTESTUse the media's own preferred size

Renditions and screen annotations

A MediaRendition ties a clip to its play parameters, and a ScreenAnnotation gives it a place on the page.

use Papier\Annotation\ScreenAnnotation;
use Papier\Multimedia\{MediaClip, MediaPlayParams, MediaRendition};

$clip   = MediaClip::fromFile('intro.mp4', 'video/mp4', 'Intro Video');
$params = (new MediaPlayParams())
    ->setVolume(90)
    ->setShowControls(true)
    ->setAutoPlay(false)
    ->setFitStyle(MediaPlayParams::FIT_MEET);

$rendition = new MediaRendition($clip, 'Intro Video');
$rendition->setPlayParams($params);

$screen = new ScreenAnnotation(72, 400, 292, 510);
$screen->setTitle('Intro Video')
       ->setRendition($rendition->getDictionary());

$page->addAnnotation($screen);

Two easy mistakes: MediaRendition's first argument is a MediaClip object (not a MIME string), and setRendition() expects a PDF object, so pass $rendition->getDictionary().

Because the screen annotation itself paints nothing until playback starts, draw a placeholder underneath it:

use Papier\Elements\{Color, Rectangle, Text};

$page->add(
    Rectangle::create(72, 400, 220, 110)->fill(Color::rgb(0.05, 0.05, 0.05)),
    Text::write('▶  intro.mp4')->at(100, 450)->font($bold, 12)->color(Color::white()),
);
$page->addAnnotation($screen);

Background audio uses the same construction with a small, unobtrusive rectangle:

$audioClip   = MediaClip::fromFile('background.mp3', 'audio/mpeg', 'Background Music');
$audioParams = (new MediaPlayParams())
    ->setVolume(60)
    ->setShowControls(false)
    ->setAutoPlay(true)
    ->setRepeatCount(0.0);   // loop forever

$audioRendition = new MediaRendition($audioClip, 'Background Music');
$audioRendition->setPlayParams($audioParams);

$audioScreen = new ScreenAnnotation(320, 435, 520, 475);
$audioScreen->setTitle('Background Music')
            ->setRendition($audioRendition->getDictionary());
$page->addAnnotation($audioScreen);

Legacy movie annotations

MovieAnnotation is the PDF 1.2 mechanism. It is simpler than a rendition but has no MIME typing and less playback control. Use it only for compatibility with older viewers.

use Papier\Annotation\MovieAnnotation;
use Papier\Multimedia\{MovieActivation, MovieDictionary};

$movie = MovieDictionary::fromFile('sample.mp4')
    ->setAspect(1280, 720)
    ->setPoster(true);        // show the first frame as a still

$activation = (new MovieActivation())
    ->setVolume(1.0)          // 0.0-1.0 here, unlike MediaPlayParams
    ->setShowControls(true)
    ->setMode(MovieActivation::MODE_ONCE);

$annot = new MovieAnnotation(72, 560, 272, 640);
$annot->setTitle('Sample Movie')
      ->setMovie($movie->getDictionary())
      ->setActivation($activation->getDictionary());

$page->addAnnotation($annot);

The activation modes are MODE_ONCE, MODE_OPEN (play and leave the window open), MODE_REPEAT and MODE_PALINDROME (play forwards then backwards). setRate() adjusts playback speed and setSynchronous() blocks the viewer until playback finishes.

Media elements

The high-level elements API wraps the common cases so you do not have to assemble the pieces or call addAnnotation() yourself. SoundElement draws a click-to-play icon and registers the sound annotation:

use Papier\Elements\{Color, SoundElement};

$page->add(
    SoundElement::create($sound, 72, 670)
        ->size(22)
        ->icon('Speaker')
        ->color(Color::rgb(0.15, 0.45, 0.8))
        ->contents('440 Hz tone, 0.5 s'),
);

VideoElement draws the placeholder rectangle and attaches the screen annotation and rendition in one step:

use Papier\Elements\VideoElement;
use Papier\Multimedia\{MediaClip, MediaPlayParams};

$clip = MediaClip::fromFile('demo.mp4', 'video/mp4', 'Demo Video');

$page->add(
    VideoElement::create($clip, 72, 490, 220, 138)
        ->bgColor(Color::rgb(0.05, 0.05, 0.05))
        ->label('▶  demo.mp4', $bold, 11)
        ->title('Demo Video')
        ->playParams(
            (new MediaPlayParams())
                ->setVolume(90)
                ->setShowControls(true)
                ->setAutoPlay(false)
                ->setFitStyle(MediaPlayParams::FIT_MEET)
        ),
);

This is the recommended path for slide decks and brochures. Drop down to the raw annotation API only when you need control the elements do not expose.
